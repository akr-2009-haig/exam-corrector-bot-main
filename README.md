# بوت تصحيح امتحانات اللغة الإنجليزية (Exam Corrector Bot)

بوت تليجرام متطور يعمل بالذكاء الاصطناعي (Google Gemini Vision) لتصحيح امتحانات اللغة الإنجليزية للطلاب الناطقين باللغة العربية تلقائياً وبدقة عالية.

## 🌟 المميزات وآلية العمل

### 👨‍🏫 سير عمل المعلم (الآدمن)
- **إضافة نموذج الإجابة:** يقوم الآدمن بإرسال صورة (أو عدة صور) لـ "نموذج الإجابة" (Answer Key).
- **الاستخراج الذكي:** يقوم الذكاء الاصطناعي باستخراج الأسئلة، الإجابات الصحيحة، وتوزيع الدرجات تلقائياً.
- **المراجعة والتفعيل:** يعرض البوت معاينة باللغة العربية للنموذج المستخرج. من خلال الضغط على زر **"تأكيد وتفعيل"**، يتم تنشيط الامتحان الحالي وإلغاء تنشيط أي امتحانات سابقة تلقائياً.
- **إشعارات تلقائية:** عند تفعيل امتحان جديد، يقوم البوت بإرسال إشعار في الخلفية لجميع الطلاب المسجلين (الذين لم يقوموا بحظر البوت)، مع إرسال تقرير للآدمن بعدد الإشعارات الناجحة.
- **إحصائيات شاملة (📊 نتائج الطلاب):** لوحة معلومات متكاملة للامتحان النشط تتضمن: عدد الطلاب/المحاولات، أعلى/أقل/متوسط الدرجات، وتوزيع الدرجات. كما يمكن للآدمن استعراض قائمة محاولات الطلاب وعرض التصحيح المفصل وصورة ورقة إجابة كل طالب.

### 👨‍🎓 سير عمل الطالب
- **تقديم الإجابات:** يرسل الطالب صورة (أو صور) لورقة إجابته.
- **التصحيح المرن:** يقوم البوت بمطابقة إجابات الطالب مع نموذج الإجابة النشط، مع مراعاة "التسامح المعتدل" (الأخطاء الإملائية البسيطة أو الإجابات المقاربة) بفضل الذكاء الاصطناعي.
- **تقرير مفصل:** يتلقى الطالب تقريراً باللغة العربية يوضح الدرجة لكل سؤال، المجموع الكلي، وملاحظات توضيحية.
- **سجل الدرجات (📊 نتائجي):** يمكن للطالب مراجعة سجل امتحاناته وأفضل درجاته في أي وقت.

### 🔄 سياسة المحاولات وإعادة الامتحان
- **محاولة واحدة فقط:** يُسمح لكل طالب بمحاولة واحدة لكل امتحان.
- **طلب محاولة جديدة:** في حالة الرغبة في إعادة الامتحان، يمكن للطالب الضغط على زر **"🙏 طلب محاولة جديدة"**. يتم إرسال الطلب للآدمن ليقرر (السماح/الرفض).
- **استبدال النتيجة:** إذا وافق الآدمن، بمجرد تقديم الطالب للمحاولة الجديدة ونجاح تصحيحها، **تُحذف النتيجة القديمة وتُستبدل بالنتيجة الجديدة** (بحيث تظهر النتيجة الجديدة فقط في الإحصائيات وسجل الطالب).

### ⚡ أداء عالي في الخلفية
- يتم تنفيذ المهام الثقيلة (مثل تحميل الصور والتواصل مع خوادم Gemini) **في الخلفية (Background)** حتى لا يتوقف البوت عن استقبال الرسائل الجديدة.
- يُسمح لكل مستخدم بعملية واحدة فقط في نفس الوقت.
- يتم تشغيل حد أقصى 3 عمليات Gemini في وقت واحد لضمان الاستقرار.
- جميع اتصالات الشبكة مزودة بمهلة زمنية (Timeout) لتجنب التعليق.

## 🛠️ أوامر البوت

| الأمر (Command) | صلاحيات الطالب | صلاحيات الآدمن |
|-----------------|----------------|----------------|
| `/start`        | القائمة الرئيسية | القائمة الرئيسية |
| `/exams`        | عرض الامتحانات المتاحة | إدارة الامتحانات النشطة |
| `/results`      | عرض سجل درجاته | عرض الإحصائيات ونتائج جميع الطلاب |
| `/history`      | — | عرض قائمة بآخر الامتحانات |
| `/students`     | — | عرض درجات طالب معين |
| `/new`          | — | إضافة امتحان جديد |
| `/test`         | — | وضع وتجربة الامتحان |
| `/cancel`       | إلغاء العملية الحالية | إلغاء العملية الحالية |
| `/skip`         | تخطي خطوة التسجيل | — |

*ملاحظة: تظهر هذه الأوامر أيضاً في قائمة ☰ في تليجرام، وتكون مخصصة للآدمن (بناءً على `ADMIN_IDS`). يتم تمييز الآدمن عن الطالب برقم المعرف (ID)؛ إرسال الآدمن لصورة يُعتبر "نموذج إجابة"، بينما إرسال الطالب لصورة يُعتبر "ورقة إجابة".*

## ⚙️ التثبيت والتشغيل المحلي

تعتمد بيئة التشغيل على [Bun](https://bun.sh/). لتشغيل البوت على الخادم أو جهازك الشخصي:

```bash
# تثبيت الحزم الاعتمادية
bun install

# نسخ ملف إعدادات البيئة
cp .env.example .env

# الآن قم بتعبئة البيانات في ملف .env

# تشغيل البوت
bun start

# أو للتشغيل في وضع التطوير (مع التحديث التلقائي)
bun run dev
```

## 🔑 متغيرات البيئة (`.env`)

يجب إعداد المتغيرات التالية في ملف `.env`:

- `BOT_TOKEN`: توكن البوت الذي تحصل عليه من [@BotFather](https://t.me/BotFather).
- `GEMINI_API_KEY`: مفتاح واجهة برمجة تطبيقات Gemini الذي تحصل عليه من [Google AI Studio](https://aistudio.google.com/apikey).
- `GEMINI_MODEL`: موديل **Vision** المستخدم لاستخراج نموذج الإجابة وتصحيح الأوراق (الافتراضي: `gemini-3.1-flash-lite` أو `gemini-1.5-flash`).
- `GEMINI_GRADING_MODEL`: (اختياري) موديل مخصص لعملية التصحيح فقط. إذا تُرك فارغاً، سيتم استخدام `GEMINI_MODEL`.
- `ADMIN_IDS`: معرفات (IDs) الآدمنز في تليجرام، مفصولة بفاصلة (مثال: `123456789,987654321`).

## 📁 الهيكلية المعمارية للمشروع

تم بناء المشروع باستخدام TypeScript و Telegraf و Bun.SQLite، وهو مقسم بالشكل التالي:

```text
├── index.ts              # نقطة الدخول، إعداد البوت، وتوجيه مسارات الصور
├── src/
│   ├── config.ts         # قراءة وإدارة متغيرات البيئة (.env)
│   ├── db.ts             # إدارة قاعدة البيانات SQLite (جداول: exams, submissions)
│   ├── gemini.ts         # التعامل مع Gemini Vision وضمان الحصول على JSON صحيح
│   ├── prompts.ts        # التلقينات (Prompts) الخاصة باستخراج النموذج والتصحيح
│   ├── format.ts         # تنسيق الرسائل باللغة العربية (معاينة النموذج، وتقارير الدرجات)
│   ├── telegram.ts       # تحميل الصور من تليجرام وتحويلها إلى Base64 Data URL
│   ├── mediaGroup.ts     # تجميع وإدارة الألبومات (عند إرسال عدة صور معاً)
│   └── handlers/         # وحدات معالجة الأوامر والرسائل
│       ├── admin.ts      # وظائف الآدمن
│       ├── student.ts    # وظائف الطالب
│       └── start.ts      # معالجة أمر البداية والتسجيل
```

> **ملاحظة حول البيانات:** يتم تخزين جميع البيانات محلياً في ملف `bot.db` باستخدام قاعدة بيانات SQLite خفيفة وسريعة عبر `bun:sqlite`.
