## 1. هوية المشروع

### الوصف العام

**بوت تصحيح امتحانات اللغة الإنجليزية** هو نظام متكامل لتصحيح الامتحانات تلقائياً عبر تليجرام، مدعوم بذكاء اصطناعي (Google Gemini Vision). يستهدف البوت المعلمين والطلاب الناطقين بالعربية، ويسمح للمعلم برفع نموذج الإجابة كصورة، ثم يقوم الذكاء الاصطناعي بتصحيح أوراق إجابات الطلاب المرسلة كصور وإرجاع تقارير تفصيلية باللغة العربية.

### الغرض الأساسي

| البُعد | التفصيل |
|--------|----------|
| **المشكلة المحلولة** | أتمتة تصحيح أوراق الامتحانات يدوياً |
| **المستخدمون** | معلمو اللغة الإنجليزية + طلابهم |
| **القناة** | بوت تليجرام + Mini App (تطبيق مدمج داخل تليجرام) |
| **اللغة الموجهة** | العربية (واجهة المستخدم والتقارير) |

## 2. المعمارية البرمجية (Architecture)

### 2.1 الصورة العامة للنظام

```
┌─────────────────────────────────────────────────────┐
│                   مستخدم تليجرام                    │
│           (معلم / طالب)                             │
└────────────────┬────────────────────────────────────┘
                 │  Telegram API
                 ▼
┌─────────────────────────────────────────────────────┐
│              بوت تليجرام (Thin Launcher)             │
│              index.ts — Telegraf                    │
│  ┌──────────────────┐   ┌─────────────────────┐    │
│  │ Channel Join Gate│   │ Outbox Drainer      │    │
│  │ (membership.ts)  │   │ (إرسال إشعارات)     │    │
│  └──────────────────┘   └─────────────────────┘    │
│                 │                                   │
│                 ▼  فتح رابط                         │
│  ┌──────────────────────────────────────────────┐   │
│  │         Mini App (Next.js)                   │   │
│  │   منطق المنتج الكامل: رفع الصور،            │   │
│  │   التصحيح، النتائج، الإحصائيات             │   │
│  └──────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────────────────┐
│           Google Gemini Vision API                  │
│   (استخراج نموذج الإجابة + تصحيح الأوراق)         │
└─────────────────────────────────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────────────────┐
│           SQLite Database (bot.db)                  │
│   جداول: users / outbox / exams / submissions       │
└─────────────────────────────────────────────────────┘
```

### 2.2 النمط المعماري

المشروع يعتمد **معمارية هجينة متطورة** تمر بمرحلتين:

| المرحلة | النمط | الوصف |
|---------|-------|-------|
| **النسخة الأولى** | Bot-Only Architecture | بوت تليجرام يحمل منطق التصحيح كاملاً داخله |
| **النسخة الحالية** | Bot + Mini App Hybrid | البوت أصبح "قاذف خفيف" (Thin Launcher) فقط، وانتقل منطق المنتج كله إلى Mini App (Next.js) |

#### مبدأ Separation of Concerns الحالي:

```
البوت (index.ts) يتولى:          Mini App يتولى:
─────────────────────             ────────────────────────
✓ بوابة عضوية القناة             ✓ رفع ومعالجة الصور
✓ تحديث جدول users              ✓ التصحيح بالذكاء الاصطناعي
✓ إرسال إشعارات Outbox          ✓ عرض النتائج والإحصائيات
✓ فتح رابط Mini App             ✓ إدارة الامتحانات
                                  ✓ قوائم الطلاب والمسابقات
```

## 4. المكونات التفصيلية (Components)

### 4.1 نقطة الدخول — `index.ts`

**الدور:** القلب النابض للبوت، يربط كل الوحدات معاً.

**المسؤوليات:**
- تهيئة بوت Telegraf مع مهلة زمنية 30 ثانية
- تسجيل Middleware لتحديث بيانات المستخدمين تلقائياً
- تطبيق بوابة عضوية القناة على كل تفاعل
- إرسال رسالة "فتح التطبيق" عبر زر Web App
- تشغيل حلقة مؤقتة لاستنزاف جدول `outbox` وإرسال الإشعارات

```typescript
// مثال على الزر المفتوح للـ Mini App
Markup.inlineKeyboard([
  [Markup.button.webApp("🚀 فتح التطبيق", miniAppUrl)]
])
```

---

### 4.2 الإعداد والتهيئة — `src/config.ts`

**الدور:** مركز قراءة جميع متغيرات البيئة وتوزيعها.

| المتغير | النوع | الوصف |
|---------|-------|-------|
| `BOT_TOKEN` | نص (مطلوب) | توكن بوت تليجرام من @BotFather |
| `ADMIN_IDS` | أرقام مفصولة بفاصلة | معرفات المشرفين |
| `MINIAPP_URL` | رابط HTTPS | رابط Mini App المدمج |
| `REQUIRED_CHANNEL_ID` | معرف قناة | القناة المطلوب الاشتراك بها (اختياري) |
| `REQUIRED_CHANNEL_USERNAME` | نص | اسم مستخدم القناة للعرض |

**الدوال المصدَّرة:**
- `config` — كائن الإعدادات الموحد
- `isAdmin(userId)` — دالة للتحقق من صلاحيات المشرف

---

### 4.3 قاعدة البيانات — `src/db.ts`

**الدور:** طبقة وصول خفيفة لقاعدة بيانات SQLite.

**التقنية:** `bun:sqlite` (مدمج في Bun، لا يحتاج تبعية خارجية)

**الإعدادات الحرجة:**
```sql
PRAGMA journal_mode = WAL;     -- أداء أعلى في القراءة/الكتابة المتزامنة
PRAGMA busy_timeout = 5000;    -- انتظار 5 ثوان عند التزاحم
```

**الجداول:**

| الجدول | الحقول الرئيسية | الوصف |
|--------|-----------------|-------|
| `users` | user_id, first_name, username, selected_exam_id | بيانات الطلاب والمعلمين |
| `outbox` | id, user_id, text, buttons_json, sent_at | قائمة انتظار الإشعارات |
| `exams` | (يديرها Mini App) | بيانات الامتحانات ونماذج الإجابة |
| `submissions` | (يديرها Mini App) | إجابات وتصحيحات الطلاب |

**ملاحظة معمارية مهمة:** البوت والـ Mini App يفتحان نفس ملف `bot.db` بشكل آمن بفضل وضع WAL. البوت يقرأ ويكتب في `users` و`outbox` فقط، أما `exams` و`submissions` فتديرهما Mini App حصراً.

**الدوال المصدَّرة:**
```typescript
upsertUser(userId, firstName, username)  // تحديث أو إنشاء مستخدم
unsentOutbox(limit?)                     // جلب الإشعارات غير المرسلة
markOutboxSent(id)                       // تعليم الإشعار كـ "مُرسَل"
```

---

### 4.4 بوابة العضوية — `src/membership.ts`

**الدور:** التحقق الإجباري من اشتراك المستخدم في قناة تليجرام قبل استخدام البوت.

**آلية العمل:**
1. كل تفاعل غير مشرف → تحقق فوري من العضوية عبر `getChatMember`
2. إذا لم يكن عضواً → إرسال رسالة بزرين: "انضم للقناة" + "تحققت"
3. ضغط زر "تحققت" → تحقق جديد فوري وفتح التطبيق

**خاصية Fail-Open:** إذا فشل الاستعلام (بوت ليس مشرف في القناة) → يُفتح للجميع ويُسجَّل تحذير في السجلات. يمنع هذا القفل التام للمستخدمين بسبب أخطاء الإعداد.

---

### 4.5 تكامل الذكاء الاصطناعي — `src/gemini.ts`

**الدور:** التواصل مع Google Gemini Vision API لاستخراج النماذج وتصحيح الأوراق.

**المهام الرئيسية:**
- إرسال صور نماذج الإجابة لاستخراج الأسئلة والإجابات والدرجات
- إرسال صور أوراق الطلاب لمقارنتها مع النموذج
- ضمان الحصول على JSON صالح من الاستجابة (JSON extraction/repair)
- دعم نموذجين منفصلين: نموذج للاستخراج، ونموذج للتصحيح

**خصائص الأداء:**
- حد أقصى **3 عمليات Gemini** متزامنة (لمنع تشبع الخدمة)
- مهلة زمنية (Timeout) على كل طلب شبكي

---

### 4.6 التلقينات — `src/prompts.ts`

**الدور:** يحتوي على نصوص التلقينات (Prompts) المرسلة لـ Gemini.

**أنواع التلقينات:**
1. **تلقين الاستخراج:** يطلب من Gemini قراءة نموذج الإجابة وإرجاع JSON يتضمن الأسئلة، الإجابات الصحيحة، وتوزيع الدرجات.
2. **تلقين التصحيح:** يطلب من Gemini مقارنة إجابات الطالب بنموذج الإجابة مع "تسامح معتدل" للأخطاء الإملائية البسيطة والإجابات المقاربة.

---

### 4.7 التنسيق والعرض — `src/format.ts`

**الدور:** تحويل البيانات الخام إلى رسائل تليجرام منسقة باللغة العربية.

**المخرجات:**
- معاينة نموذج الإجابة المستخرج (للمشرف قبل التأكيد)
- تقرير تصحيح الطالب (درجة لكل سؤال + المجموع + ملاحظات)
- رسائل الحالة والأخطاء

---

### 4.8 معالجة الصور — `src/telegram.ts` و `src/mediaGroup.ts`

| الملف | الوظيفة |
|-------|---------|
| `telegram.ts` | تحميل الصور من خوادم تليجرام وتحويلها إلى Base64 Data URL لإرسالها لـ Gemini |
| `mediaGroup.ts` | تجميع وإدارة ألبومات الصور (عند إرسال أكثر من صورة في رسالة واحدة)، وذلك لأن تليجرام يرسل كل صورة في ألبوم كرسالة منفصلة |

---

### 4.9 معالجات الأوامر — `src/handlers/admin.ts`

**الدور:** تنفيذ الوظائف المتاحة حصراً للمشرفين.

**الوظائف:**
- عرض نماذج الإجابة والتأكيد عليها
- استعراض إحصائيات الامتحانات
- عرض تصحيح الطلاب الفردي
- الموافقة أو رفض طلبات إعادة الامتحان

## 10. القرارات المعمارية الحرجة

### 10.1 لماذا Bun بدلاً من Node.js؟
- SQLite مدمج في Bun (`bun:sqlite`) — لا حاجة لتبعية خارجية
- تشغيل TypeScript مباشرة بدون خطوة بناء في التطوير
- أداء أعلى وبدء تشغيل أسرع

### 10.2 لماذا الانتقال إلى نموذج Bot + Mini App؟
- **حدود تليجرام:** البوت التقليدي محدود في تجربة المستخدم (نص فقط)
- **Mini App:** واجهة ويب كاملة داخل تليجرام (مثل متصفح مصغر)
- **التوزيع الأمثل:** البوت يتخصص بالإشعارات والبوابة، أما المنطق المعقد فيذهب للـ Mini App

### 10.3 نمط Outbox لإرسال الإشعارات
- Mini App يكتب الإشعارات في جدول `outbox`
- البوت يقرأ من `outbox` ويرسلها عبر تليجرام
- هذا يفصل منطق البيزنس (Mini App) عن قناة الإشعارات (Bot) بشكل نظيف

### 10.4 WAL Mode في SQLite
```sql
PRAGMA journal_mode = WAL;
```
يسمح بالقراءة المتزامنة من عملية Bot وعملية Mini App على نفس الملف دون تعارض.

### 10.5 Fail-Open في بوابة العضوية
عند فشل استعلام `getChatMember` (خطأ في الإعداد) → يُتاح الوصول للجميع مع تسجيل تحذير. أفضل من قفل جميع المستخدمين بسبب خطأ إعداد.

## 11. نقاط القوة والملاحظات

### نقاط القوة
- ✅ معمارية نظيفة بفصل واضح للمسؤوليات
- ✅ التسامح مع الأخطاء الإملائية في التصحيح (بفضل Gemini)
- ✅ دعم الألبومات (رفع عدة صور للامتحان الواحد)
- ✅ نظام إشعارات موثوق عبر نمط Outbox
- ✅ تحديد النمط المعماري بوضوح (Bot = Launcher فقط)
- ✅ معالجة الأخطاء مع Fail-Open لضمان الاستمرارية

### ملاحظات للتطوير
- ⚠️ ملف `gemini.ts` في المستودع يبدو قديماً أو محذوفاً (لم يكن متاحاً عند التحليل) — منطق Gemini ربما انتقل للـ Mini App
- ⚠️ لا يوجد توثيق لـ Mini App (Next.js) ضمن هذا المستودع — قد يكون مستودعاً منفصلاً
- ⚠️ SQLite مناسب للنطاق الحالي؛ عند التوسع الكبير قد يُنظر في الانتقال لـ PostgreSQL
